# inote 需求文档

## 1. 项目概述

inote 是一个面向个人用户的知识库问答系统。用户登录后，可以上传本人的文档，系统对文档进行解析、切分、向量化与索引构建，并基于检索增强生成（RAG）能力发起问答。系统同时提供会话管理、来源引用展示、密码重置和文档状态跟踪能力。

当前实现为前后端分离架构：

- 后端：Spring Boot 3 + JPA + PostgreSQL + Spring AI + Milvus
- 前端：React + TypeScript + Vite

## 2. 建设目标

- 为用户提供基于私有文档的问答能力。
- 确保不同用户之间的数据隔离，用户只能访问自己的文档和会话。
- 支持文档上传后的异步处理与状态追踪。
- 在回答中返回可访问的来源文档链接。
- 提供基础的账户认证能力，包括验证码登录、首次自动注册和密码重置。

## 3. 用户角色

当前系统仅实现一种业务角色：

- 普通登录用户：可管理自己的文档、会话和问答记录。

系统未实现管理员端、组织级角色、共享空间或多级权限模型。

## 4. 业务范围

### 4.1 已实现范围

- 用户验证码登录与首次自动注册
- 当前登录态识别
- 密码重置
- 文档上传、解析、入库、状态查看、文件访问
- 基于个人知识库的问答
- 对话会话创建、查看、重命名、删除
- 回答来源引用展示
- 前端工作台式交互界面

### 4.2 未实现范围

- 管理员后台
- 用户邀请、组织空间、协作共享
- 文档删除、文档重命名、文档标签管理
- 流式回答
- 会话导出
- OCR、图片文档解析
- 细粒度审计报表
- 正式验证码图片或短信/邮件验证

## 5. 核心业务规则

- 所有受保护接口通过请求头 `X-Auth-Token` 识别当前用户。
- 若用户名首次登录且验证码正确，系统自动创建账户。
- 用户名会被标准化为小写。
- 用户只能访问自己的文档、会话与文件下载地址。
- 当问题未携带 `sessionId` 时，系统仅返回回答，不自动保存会话记录。
- 当问题携带 `sessionId` 时，系统会保存用户消息与助手消息各一条。
- 新建会话默认标题为 `New Session`。
- 默认标题的会话在首轮问答后，会自动截取首个问题生成摘要标题。
- 回答必须以检索到的参考文档为依据；未检索到相关文档时，返回“当前文档不足以回答问题”类固定提示。
- 回答返回的来源文档最多保留 3 条去重结果。

## 6. 功能需求

### 6.1 用户认证

#### 6.1.1 获取验证码

- 系统应提供验证码获取接口。
- 验证码应包含 `captchaId` 和明文 `captchaCode`。
- 验证码默认有效期为 5 分钟。
- 验证码长度默认取配置值，且最短不低于 4 位。

#### 6.1.2 登录 / 自动注册

- 用户提交用户名、密码、验证码 ID 和验证码内容后，系统应完成认证。
- 若用户名不存在，系统应自动注册并返回登录态令牌。
- 若用户名已存在，系统应校验密码。
- 验证码校验失败时，应拒绝登录。
- 登录成功后，系统应刷新并返回新的认证令牌。

#### 6.1.3 当前用户信息

- 系统应支持基于当前令牌查询当前用户信息。
- 返回信息至少包括用户名和当前令牌。

#### 6.1.4 密码重置

- 登录用户应可提交新密码与确认密码完成密码重置。
- 两次密码输入必须一致。
- 新密码长度不得少于 6 位。
- 密码重置成功后，系统应刷新认证令牌。

### 6.2 文档管理

#### 6.2.1 上传文档

- 用户应可上传一个或多个文档。
- 后端单文件大小限制为 50MB，请求总大小限制为 50MB。
- 支持的文件类型为：
  - PDF
  - DOC
  - DOCX
  - XLS
  - XLSX
  - TXT
  - CSV
- 空文件、无扩展名非法文件、非支持格式文件应被拒绝。
- 上传成功后，系统应立即返回文档 ID、文件名、状态和处理提示信息。

#### 6.2.2 文档处理流程

- 文档上传后应进入异步处理流程。
- 当前实现中的状态至少包括：
  - `PARSING`
  - `COMPLETED`
  - `FAILED`
- 前端将 `PENDING`、`PARSING`、`PROCESSING` 视为处理中状态集合，需求上应兼容处理中状态扩展。
- 处理流程应包括：
  - 文件落盘保存
  - 文本解析
  - 文本分块
  - 向量嵌入与向量库存储
  - BM25 索引构建
- 任一步骤失败时，文档状态应标记为失败，并记录错误信息。

#### 6.2.3 文档列表与状态查看

- 用户应可查看自己的文档列表。
- 列表项至少包括：
  - 文档 ID
  - 文件名
  - 文件大小
  - 状态
  - 错误信息
  - 更新时间
- 用户应可查看单个文档状态。

#### 6.2.4 文档文件访问

- 用户应可通过文档 ID 获取原始文件访问地址。
- 系统必须校验文件归属权。
- 系统必须阻止访问上传目录之外的文件路径。

### 6.3 智能问答

#### 6.3.1 问答输入

- 用户应可输入自然语言问题发起问答。
- 请求参数至少包括：
  - `question`
  - 可选的 `sessionId`
- 问题内容不能为空。

#### 6.3.2 检索流程

- 系统应基于当前用户自己的文档执行检索。
- 当前默认启用向量检索。
- 系统预留以下可配置能力，但默认可关闭：
  - 查询改写
  - 多路子查询分解
  - 混合检索
  - 重排
- 当前检索返回数量、相似度阈值、RRF 参数、重排 TopN 等应支持配置。

#### 6.3.3 回答生成

- 系统应使用检索结果构造提示词并调用大模型生成回答。
- 回答生成应结合最近最多 20 条历史消息理解上下文。
- 若检索结果为空，系统应直接返回固定兜底文案。
- 主模型调用失败时，系统应自动切换备用模型。
- 当主模型与备用模型都失败时，系统应返回服务暂不可用提示。

#### 6.3.4 来源引用

- 回答结果中应返回来源列表。
- 每个来源至少包括：
  - 文件名
  - 文件访问 URL
- 来源列表应按检索结果去重，最多返回 3 条。

### 6.4 会话管理

#### 6.4.1 新建会话

- 用户应可新建会话。
- 若未传标题，则默认标题为 `New Session`。

#### 6.4.2 会话列表

- 用户应可查看自己的会话列表。
- 列表项至少包括：
  - 会话 ID
  - 标题
  - 消息数量
  - 创建时间
  - 更新时间

#### 6.4.3 会话详情

- 用户应可查看单个会话详情。
- 会话详情应返回完整消息列表。
- 消息项至少包括：
  - 消息 ID
  - 角色
  - 内容
  - 创建时间

#### 6.4.4 会话编辑

- 用户应可修改会话标题。
- 用户应可删除自己的会话。
- 删除会话时，应同时删除其下属消息记录。

### 6.5 前端工作台

前端当前应提供以下页面能力：

- 登录页
  - 用户名输入
  - 密码输入
  - 验证码展示与刷新
  - 登录 / 自动注册提交
- 工作台首页
  - 会话侧边栏
  - 当前会话消息区
  - 文档知识库侧栏
  - 最近引用展示
  - 新建会话
  - 会话搜索
  - 会话固定
  - 会话重命名
  - 会话删除
  - 文件上传
  - 文档状态刷新
  - 密码重置入口
  - 退出登录
- 前端应在存在处理中文档时，每 3 秒自动轮询文档状态。
- 前端应将认证令牌和固定会话 ID 列表保存在浏览器本地存储中。

## 7. 数据与状态要求

### 7.1 用户数据

- 用户实体至少包含：
  - 用户 ID
  - 用户名
  - 密码哈希
  - 认证令牌
  - 创建时间
  - 更新时间

### 7.2 文档数据

- 文档实体至少包含：
  - 文档 ID
  - 文件名
  - 文件路径
  - 文件访问 URL
  - 所属用户
  - 内容类型
  - 文件大小
  - 状态
  - 错误信息
  - 创建时间
  - 更新时间

### 7.3 会话数据

- 会话实体至少包含：
  - 会话 ID
  - 标题
  - 所属用户
  - 创建时间
  - 更新时间

### 7.4 消息数据

- 消息实体至少包含：
  - 消息 ID
  - 所属会话
  - 角色
  - 内容
  - 创建时间

## 8. 非功能需求

### 8.1 安全性

- 系统必须基于令牌识别当前用户。
- 所有用户数据访问必须按用户归属校验。
- 越权访问他人会话或文档时，应返回未找到，而非泄露资源存在性。
- 文件访问必须限制在上传目录内。
- 密码必须以哈希形式存储，不得明文保存。

### 8.2 可用性

- 文档处理应异步执行，避免上传接口长时间阻塞。
- 模型调用应具备主备切换容错能力。
- 系统异常时应返回统一错误结构或可识别错误消息。

### 8.3 可配置性

- 以下能力应支持通过配置文件或环境变量调整：
  - 服务端口
  - 数据库连接
  - 上传目录
  - 上传大小限制
  - 主模型与备用模型参数
  - Milvus 连接参数
  - RAG 检索参数
  - 验证码长度

## 11. 验收要点

- 未注册用户可通过正确验证码完成首次登录并自动注册。
- 已注册用户使用正确用户名、密码、验证码可登录成功。
- 错误密码、错误验证码、过期验证码会被拒绝。
- 登录后只能访问自己的文档与会话。
- 上传支持格式文件后，系统可生成文档记录并进入处理流程。
- 上传超限或不支持格式文件时，系统返回明确错误。
- 已处理完成的文档可参与问答检索。
- 未检索到相关内容时，系统返回固定兜底回答。
- 回答结果包含来源文档链接。
- 可完成会话的新建、查看、重命名、删除。
- 首轮问答后默认新会话标题可自动更新为问题摘要。
- 登录用户可重置密码，重置后旧令牌失效，后续应使用新令牌。
