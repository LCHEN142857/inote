# inote 项目规格说明

本文档描述当前仓库已实现的产品范围、业务规则、接口能力与验收口径。它面向需求确认、开发排期和测试验收；开发启动与代码定位请参考 `README.md`。

## 1. 项目概述

inote 是一个面向登录用户的个人文档知识库问答系统。用户可以上传自己的文档，系统对文档进行解析、切块、向量化和索引构建，并基于检索增强生成（RAG）能力回答用户问题。系统同时提供验证码登录/自动注册、会话管理、文档状态跟踪、引用来源展示、密码重置和回答模式设置。

当前实现为前后端分离架构：

| 层 | 当前实现 |
| --- | --- |
| 前端 | React 18 + TypeScript + Vite |
| 后端 | Spring Boot 3.4.1 + Java 21 + JPA |
| 数据库 | PostgreSQL 运行时，H2 测试 |
| RAG | Spring AI OpenAI 兼容接口 + Milvus + Lucene BM25 |
| 文件解析 | PDFBox、Apache POI、纯文本读取 |

当前仓库未体现 Redis、消息队列、管理员后台、组织空间、Docker 部署文件和 CI/CD 脚本。

## 2. 建设目标

1. 为每个登录用户提供隔离的个人知识库问答能力。
2. 支持常见办公文档上传后的异步解析、入库和状态跟踪。
3. 在回答中展示可追溯的来源文档和片段预览。
4. 支持多轮会话管理，包括新建、查看、重命名、删除和本地置顶。
5. 支持通过配置切换模型、调整 RAG 检索参数和外部依赖地址。
6. 在主模型不可用时提供备用模型容错。

## 3. 用户角色

当前系统仅实现一种业务角色：

| 角色 | 能力 |
| --- | --- |
| 普通登录用户 | 管理自己的文档、会话、问答记录、密码和回答模式设置 |

未实现管理员、租户、组织、团队协作、共享知识库或多级权限模型。

## 4. 业务范围

### 4.1 已实现范围

- 验证码登录。
- 首次登录自动注册。
- 当前登录态识别。
- 登录失败锁定。
- 密码重置。
- 用户回答模式设置：仅根据参考文档回答 / 允许通用知识补充。
- 文档上传、解析、入库、状态查看。
- 同名文档版本处理：同一用户上传同名新文档后旧版本标记为非活跃。
- 失败文档删除。
- 原始文件受控访问。
- 基于个人知识库的非流式问答。
- 基于个人知识库的 SSE 流式问答。
- 聊天模型列表获取和前端模型选择。
- 会话创建、列表、详情、重命名、删除。
- 回答来源引用展示和最近引用统计。

- 文档重命名、标签、分类、目录管理。
- 会话导出。
- OCR 和图片文档解析。
- 文档处理进度百分比。

## 5. 核心业务规则

1. 除验证码和登录接口外，所有 `/api/v1/**` 业务接口都必须通过请求头 `X-Auth-Token` 识别当前用户。
2. 用户名登录时会被标准化为小写。
3. 用户名规则为 3-32 位字母、数字、下划线或短横线。
4. 密码长度不得少于 6 位。
5. 验证码为一次性验证码，有效期 5 分钟。
6. 用户名首次登录且验证码正确时，系统自动创建账号。
7. 登录成功后系统刷新并返回新的认证 token。
8. 用户只能访问自己拥有的文档、会话、消息和文件资源。
9. 越权访问其他用户的文档或会话时，业务层表现为资源不存在。
10. 问答请求携带 `sessionId` 时，系统保存用户消息和助手消息；不携带 `sessionId` 时仅返回回答，不保存会话消息。
11. 新建会话默认标题为 `New Session`；首轮问答后，如标题仍为默认值，会截取首个问题生成摘要标题。
12. 回答来源最多返回 3 条按 URL 去重后的来源。
13. 用户设置为“仅根据参考文档回答”时，未检索到相关文档将直接返回固定兜底文案。
14. 用户关闭“仅根据参考文档回答”时，未检索到相关文档可调用通用模型回答，并在提示词中要求区分通用知识和上传文档内容。
15. 上传同名文档时，新文档保持 `active=true`，同一用户同名旧文档设为 `active=false`，非活跃文档不参与检索。
16. 文件访问必须校验所属用户，并限制文件路径在上传目录内。

## 6. 功能需求

### 6.1 用户认证

#### 6.1.1 获取验证码

- 系统提供验证码获取接口。
- 响应包含 `captchaId` 和 `captchaImage`。
- `captchaImage` 为前端可直接展示的图片数据。
- 验证码有效期为 5 分钟。
- 验证码内容不应以明文返回给前端。

#### 6.1.2 登录 / 自动注册

- 用户提交用户名、密码、验证码 ID、验证码内容后完成登录。
- 用户不存在时，系统自动注册并返回登录 token。
- 用户存在时，系统校验密码。
- 验证码错误、过期或已使用时应拒绝登录。
- 登录失败会记录失败次数，并在达到阈值后短时间锁定。
- 登录成功后清除登录锁定状态并刷新 token。

#### 6.1.3 当前用户

- 系统支持根据当前 token 查询登录用户。
- 响应至少包含 `username` 和 `token`。

#### 6.1.4 密码重置

- 登录用户可提交新密码和确认密码重置密码。
- 两次密码必须一致。
- 新密码至少 6 位。
- 密码重置后刷新 token。

#### 6.1.5 用户设置

- 登录用户可查询当前回答模式设置。
- 登录用户可更新 `answerFromReferencesOnly`。
- 默认行为为仅根据参考文档回答。

### 6.2 文档管理

#### 6.2.1 上传文档

- 用户可一次选择一个或多个文档，前端逐个调用上传接口。
- 后端单文件大小限制为 50MB，请求总大小限制为 50MB。
- 当前后端支持扩展名：
  - `pdf`
  - `docx`
  - `xlsx`
  - `xls`
  - `txt`
  - `csv`
- 当前后端不支持 `doc`。
- 空文件、无文件名、无扩展名或不支持扩展名的文件应被拒绝。
- 上传成功后立即返回文档 ID、文件名、状态和提示信息。
- 上传成功初始状态为 `PARSING`。

#### 6.2.2 文档处理流程

文档上传后进入异步处理流程：

1. 保存原始文件到 `file.upload.path`。
2. 创建数据库文档记录。
3. 标记同一用户同名旧文档为非活跃。
4. 根据文件类型解析纯文本。
5. 将文本按 1000 字符切块，保留 200 字符重叠。
6. 写入 Milvus 向量库。
7. 写入进程内 Lucene BM25 索引。
8. 成功后状态改为 `COMPLETED`。
9. 失败后状态改为 `FAILED`，并记录错误信息。

当前文档状态至少包括：

| 状态 | 说明 |
| --- | --- |
| `PARSING` | 已接收并正在解析/入库 |
| `COMPLETED` | 处理完成，可参与检索 |
| `FAILED` | 处理失败，可在前端删除 |

前端将 `PENDING`、`PARSING`、`PROCESSING` 视为处理中状态，以兼容后续扩展。

#### 6.2.3 文档列表与状态

- 用户可查看自己的文档列表。
- 列表按更新时间倒序返回。
- 列表项包含：
  - `documentId`
  - `fileName`
  - `fileSize`
  - `status`
  - `active`
  - `errorMessage`
  - `updatedAt`
- 用户可查看单个文档状态。
- 前端在存在处理中状态文档时，每 3 秒轮询文档列表。

#### 6.2.4 文件访问

- 用户可通过文档 ID 访问自己拥有的原始文件。
- 后端返回 inline 文件资源。
- 后端必须校验文档归属。
- 后端必须阻止访问上传目录以外的文件路径。

#### 6.2.5 失败文档删除

- 用户可删除自己名下 `FAILED` 状态文档。
- 非 `FAILED` 状态文档不得通过当前接口删除。
- 删除失败文档时应同步尝试删除磁盘文件。

### 6.3 智能问答

#### 6.3.1 问答输入

- 用户可输入自然语言问题发起问答。
- 请求参数包含：
  - `question`
  - 可选 `sessionId`
  - 可选 `model`
- 问题内容不能为空。
- 模型名必须在后端配置的可用模型列表内；不合法时使用默认模型。

#### 6.3.2 检索流程

- 检索范围必须限定为当前用户自己的文档。
- 只有 `active=true` 的文档参与检索。
- 当前检索流水线支持：
  - 查询改写。
  - 多查询拆分。
  - Milvus 向量检索。
  - Lucene BM25 检索。
  - RRF 融合。
  - 可选重排。
- 检索参数通过 `rag.*` 配置：
  - `query-rewrite-enabled`
  - `multi-query-enabled`
  - `max-sub-queries`
  - `hybrid-search-enabled`
  - `vector-top-k`
  - `bm25-top-k`
  - `similarity-threshold`
  - `rrf-k`
  - `rerank-enabled`
  - `rerank-model`
  - `rerank-top-n`
  - `final-top-k`

#### 6.3.3 回答生成

- 系统使用检索结果构造提示词并调用大模型生成答案。
- 回答生成结合最近最多 20 条历史消息理解上下文。
- 有参考文档时，应基于参考文档生成回答。
- 无参考文档且用户设置为仅根据参考文档回答时，应直接返回固定兜底提示。
- 无参考文档且用户允许通用知识回答时，可调用通用模型回答。
- 主模型调用失败时，应自动尝试备用模型。
- 主模型和备用模型都失败时，应返回服务暂不可用提示。

#### 6.3.4 流式回答

- 系统支持 SSE 流式问答接口。
- 流式事件类型包括：
  - `metadata`：返回 `sessionId` 和来源列表。
  - `delta`：返回增量文本。
  - `done`：返回完整答案和来源列表。
- 前端当前主要使用流式接口。
- 非流式问答接口仍保留。

#### 6.3.5 来源引用

- 回答结果应返回来源列表。
- 每个来源包含：
  - `fileName`
  - `url`
  - 可选 `preview`
- 来源按 URL 去重，最多返回 3 条。
- 前端展示来源文件名和预览片段，并统计最近被引用文档。

### 6.4 会话管理

#### 6.4.1 新建会话

- 用户可新建会话。
- 可选传入标题。
- 未传标题时，标题为 `New Session`。

#### 6.4.2 会话列表

- 用户可查看自己的会话列表。
- 会话按更新时间倒序返回。
- 列表项包含：
  - `id`
  - `title`
  - `messageCount`
  - `createdAt`
  - `updatedAt`

#### 6.4.3 会话详情

- 用户可查看单个会话详情。
- 会话详情返回完整消息列表。
- 消息项包含：
  - `id`
  - `role`
  - `content`
  - `createdAt`
  - 可选 `sources`

#### 6.4.4 会话编辑

- 用户可修改自己会话的标题。
- 用户可删除自己的会话。
- 删除会话时应级联删除其下属消息。
- 前端支持本地置顶会话，置顶信息仅保存在浏览器 `localStorage`，后端不持久化。

### 6.5 前端工作台

前端当前提供两个逻辑页面：

| 页面 | 路径 | 能力 |
| --- | --- | --- |
| 登录页 | `/login` | 用户名、密码、验证码展示/刷新、登录/自动注册 |
| 工作台 | `/index` | 会话列表、聊天、模型选择、回答模式开关、文档上传、状态查看、最近引用、密码重置、退出 |

前端应具备以下交互能力：

- 登录成功后保存 token 到 `localStorage`。
- 请求自动携带 `X-Auth-Token`。
- token 失效或接口返回 401 时清理本地登录态并回到登录页。
- 使用 Vite 代理将 `/api` 转发到后端 `http://localhost:8080`。
- 支持 Enter 发送、Shift + Enter 换行。
- 上传多个文件时逐个调用上传接口。
- 有处理中状态文档时自动轮询。
- 会话可搜索、置顶、重命名、删除。
- 可选择后端返回的聊天模型。
- 可切换“仅根据参考文档回答”。

## 7. 数据与状态要求

### 7.1 用户数据

`User` 实体至少包含：

- `id`
- `username`
- `passwordHash`
- `authToken`
- `answerFromReferencesOnly`
- `createdAt`
- `updatedAt`

### 7.2 文档数据

`Document` 实体至少包含：

- `id`
- `fileName`
- `filePath`
- `fileUrl`
- `owner`
- `contentType`
- `fileSize`
- `status`
- `active`
- `errorMessage`
- `createdAt`
- `updatedAt`

### 7.3 会话数据

`ChatSession` 实体至少包含：

- `id`
- `title`
- `owner`
- `messages`
- `createdAt`
- `updatedAt`

### 7.4 消息数据

`ChatMessage` 实体至少包含：

- `id`
- `session`
- `role`
- `content`
- `sourcesJson`
- `createdAt`

### 7.5 外部索引数据

向量库和 BM25 索引文档元数据应包含：

- `document_id`
- `file_name`
- `file_url`
- `content_type`
- `owner_id`
- `chunk_size`

注意：BM25 当前使用进程内内存索引，服务重启后不会自动从数据库重建。

## 8. 接口规格

### 8.1 认证接口

| Method | Path | 鉴权 | 说明 |
| --- | --- | --- | --- |
| GET | `/api/v1/auth/captcha` | 否 | 获取验证码 |
| POST | `/api/v1/auth/login` | 否 | 登录或自动注册 |
| GET | `/api/v1/auth/me` | 是 | 当前用户 |
| POST | `/api/v1/auth/password/reset` | 是 | 重置密码 |
| GET | `/api/v1/auth/settings` | 是 | 获取用户回答模式 |
| POST | `/api/v1/auth/settings` | 是 | 更新用户回答模式 |

### 8.2 聊天与会话接口

| Method | Path | 鉴权 | 说明 |
| --- | --- | --- | --- |
| GET | `/api/v1/chat/models` | 是 | 获取默认模型和可用模型列表 |
| POST | `/api/v1/chat/query` | 是 | 非流式问答 |
| POST | `/api/v1/chat/query/stream` | 是 | SSE 流式问答 |
| POST | `/api/v1/chat/sessions` | 是 | 新建会话 |
| GET | `/api/v1/chat/sessions` | 是 | 会话摘要列表 |
| GET | `/api/v1/chat/sessions/{sessionId}` | 是 | 会话详情 |
| PUT | `/api/v1/chat/sessions/{sessionId}` | 是 | 重命名会话 |
| DELETE | `/api/v1/chat/sessions/{sessionId}` | 是 | 删除会话 |

### 8.3 文档接口

| Method | Path | 鉴权 | 说明 |
| --- | --- | --- | --- |
| POST | `/api/v1/documents/upload` | 是 | 上传文档 |
| GET | `/api/v1/documents` | 是 | 当前用户文档列表 |
| GET | `/api/v1/documents/{documentId}` | 是 | 单个文档状态 |
| DELETE | `/api/v1/documents/delete/{documentId}` | 是 | 删除失败文档 |
| GET | `/api/v1/documents/files/{documentId}` | 是 | 获取原始文件资源 |

### 8.4 通用错误

| 状态码 | 场景 |
| --- | --- |
| 400 | 参数错误、验证码错误、不支持文件类型、非法业务请求 |
| 401 | 未登录或 token 无效 |
| 404 | 资源不存在或不属于当前用户 |
| 409 | 重复提交保护命中 |
| 413 | 上传文件超过大小限制 |
| 423 | 登录失败次数过多，账号/IP 短时锁定 |
| 429 | 请求过于频繁 |
| 500 | 未预期服务端错误、文件保存失败等 |

## 10. 非功能需求

### 10.1 安全性

- 密码必须以 hash 形式存储。
- API 必须按当前用户做数据归属校验。
- 文件访问必须限制在上传目录内。
- 登录、写操作等接口应具备限流和重复提交保护。
- 响应不应泄露其他用户资源是否存在。
- 配置文件不应包含真实密钥。

### 10.2 可用性

- 文档处理应异步执行，上传接口不应等待完整解析完成。
- 模型调用应具备主备容错。
- 错误响应应保持稳定、前端可识别。
- 前端在 token 失效时应自动回到登录页。

### 10.3 可维护性

- DTO 与前端 `types.ts` 应保持一致。
- 新增接口时应同步更新 Controller、前端 `api.ts`、测试和文档。
- 修改检索策略时应补充或调整检索服务测试。
- 生产数据库结构演进应引入迁移工具；当前仓库未体现。

### 10.4 性能与容量

- 单文件上传上限当前为 50MB。
- 文档切块大小当前为 1000 字符，重叠 200 字符。
- 流式问答 SSE 超时时间当前为 180 秒。
- BM25 当前为内存索引，不适合作为跨重启持久检索能力。

## 11. 外部依赖与运行条件

运行当前系统至少需要：

## 12. 测试与验收要点

### 12.1 认证验收

- 未注册用户可通过正确验证码首次登录并自动注册。
- 已注册用户使用正确密码可登录。
- 错误密码、错误验证码、过期验证码应被拒绝。
- 登录失败达到阈值后应返回锁定信息。
- 登录后 `/api/v1/auth/me` 能返回当前用户。
- 密码重置后旧 token 应失效或需要使用新 token。

### 12.2 文档验收

- 支持格式文件上传后生成文档记录并进入 `PARSING`。
- 处理成功后状态变为 `COMPLETED`。
- 不支持格式、空文件、超限文件应返回明确错误。
- 同名新文档上传后旧文档应变为非活跃。
- `FAILED` 状态文档可以删除，非 `FAILED` 文档不可通过当前删除接口删除。
- 用户不能访问其他用户的文档列表、状态或原始文件。

### 12.3 问答验收

- 已完成处理的文档可参与检索。
- 回答应包含来源列表，来源最多 3 条。
- 流式问答应依次产生 `metadata`、`delta`、`done` 事件。
- 携带 `sessionId` 的问答应保存用户消息和助手消息。
- 未携带 `sessionId` 的问答不应保存到会话。
- 无检索结果且仅文档回答开启时，应返回固定兜底文案。
- 主模型异常时应尝试备用模型；主备均失败时返回服务不可用提示。

### 12.4 会话验收

- 用户可新建、查看、重命名、删除自己的会话。
- 会话列表应返回消息数量。
- 删除会话应级联删除消息。
- 默认标题会话首轮问答后可自动更新为问题摘要。
- 用户不能访问或操作其他用户的会话。

### 12.5 前端验收

- 登录页可展示和刷新验证码。
- 登录成功后进入工作台。
- token 失效后前端清理状态并返回登录页。
- 工作台可上传文档、查看状态、选择模型、切换回答模式、发送流式问题。
- 有处理中状态文档时前端应自动轮询刷新。

## 13. 后续建议

后续迭代建议优先级：

1. 清理配置文件中的明文密钥，改为环境变量或密钥管理。
2. 增加正式的数据库迁移工具。
3. 实现完整文档删除，并同步清理 Milvus 和 BM25 索引。
4. 增加 BM25 索引重建能力或持久化索引方案。
5. 增加文档处理进度和失败原因前端展示。
6. 增加前端单元测试或 E2E 测试。
7. 增加 Docker Compose，统一 PostgreSQL、Milvus 和应用启动。
8. 评估组织空间、共享知识库和权限模型。
